# Arduino Library Requirements for LifeTrac v25

Active firmware lives under `DESIGN-CONTROLLER/firmware/`. The archived WiFi/BLE/MQTT Opta and ESP32 remote sketches in `RESEARCH-CONTROLLER/` are not part of the v25 hardware-test path.

## Active Arduino Targets

### `firmware/handheld_mkr/handheld.ino`
- Board: Arduino MKR WAN 1310
- Libraries:
    - RadioLib
    - SPI (board core)
    - Wire (board core, drives the SSD1306 OLED + future Qwiic peripherals)
    - Adafruit_SSD1306 (panel status display; init failure is non-fatal)
    - Adafruit_GFX (transitive dep of Adafruit_SSD1306)

### `firmware/tractor_h7/tractor_m7.ino` and `tractor_m4.cpp`
- Board: Portenta X8 onboard STM32H747 co-MCU / Portenta H7-compatible core
- Libraries:
    - RadioLib
    - ArduinoRS485
    - ArduinoModbus
    - SPI (board core)

### `firmware/tractor_opta/opta_modbus_slave.ino`
- Board: Arduino Opta WiFi
- Libraries:
    - ArduinoRS485
    - ArduinoModbus
    - OptaController (needed when the D1608S/A0602 placeholder shims are replaced with real expansion-bus calls)

### `firmware/bench/lora_retune_bench/lora_retune_bench.ino`
- Board: MKR WAN 1310 or Portenta Max Carrier/H747, selected with `LIFETRAC_BOARD_MKR_WAN1310` or `LIFETRAC_BOARD_PORTENTA_MAX`
- Libraries:
    - RadioLib

## Intentionally Not Required In Active Firmware

- WiFi
- ArduinoBLE
- PubSubClient
- ArduinoJson
- SparkFun Qwiic Joystick Arduino Library

Those belonged to the archived Opta WiFi/MQTT/BLE and ESP32 remote designs. MQTT remains in v25 only as base-station-internal Python IPC on the X8, not as an Arduino wireless control transport.

## Arduino IDE Installation

Install these libraries with Tools -> Manage Libraries:

- `RadioLib`
- `ArduinoRS485`
- `ArduinoModbus`
- `Adafruit SSD1306` (handheld OLED)
- `Adafruit GFX Library` (handheld OLED)
- `Crypto` by `rweather` (handheld AES-128-GCM via `GCM<AES128>`; production builds with `-DLIFETRAC_USE_REAL_CRYPTO`. Portenta H7 uses MbedTLS already shipped in the mbed-os core, no extra library.)
- `OptaController`

Then select the target board and run Verify on the sketch folder under `firmware/`.

## Arduino CLI Examples

```bash
arduino-cli core update-index
arduino-cli core install arduino:samd
arduino-cli core install arduino:mbed_opta
arduino-cli core install arduino:mbed_portenta

arduino-cli lib install RadioLib
arduino-cli lib install ArduinoRS485
arduino-cli lib install ArduinoModbus
arduino-cli lib install OptaController

arduino-cli compile --fqbn arduino:samd:mkrwan1310 LifeTrac-v25/DESIGN-CONTROLLER/firmware/handheld_mkr
arduino-cli compile --fqbn arduino:mbed_opta:opta LifeTrac-v25/DESIGN-CONTROLLER/firmware/tractor_opta
```

The exact FQBN for the X8 onboard H747 compile should be verified against the installed Arduino core before enabling it as a required CI gate.
